# Time Log

**Domain:** consulting / project time tracking

**Patron-style sentence:**

> Log my time on projects so I can invoice clients accurately.

**Why these entities, not others:**

Time-tracking apps fail by overcomplicating the entry form. The smallest useful entry is (project, date, hours, description) — everything else (billable, invoiced, tags) is optional metadata that lets the patron run reports. Billable + invoiced together give the consultant the "what do I owe a client an invoice for" view that is the whole point.
