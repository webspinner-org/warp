# Recipe Box

**Domain:** home cooking

**Patron-style sentence:**

> I want to organize my recipes — ingredients, steps, where I found them, what I've actually cooked.

**Why these entities, not others:**

Recipes have ingredients as a sub-entity (one-to-many) so the patron can scale them, swap items, or pull a shopping list across recipes later. The cook log captures the "did this actually work" feedback that improves the collection over time. Cuisine + tags enable filtering, which is the main UX of a recipe collection.
