### 3b. From try (Your Work modal)

1. In try's Your Work modal, find a published row and click **⬇
   Download**.
2. Same .html file downloads.

### 3c. Open the downloaded file

The file is truly self-contained — Svelte runtime + renderer + bundle
are all inlined. Zero CDN dependency. Three ways to test:

**3c-i. Direct from disk (`file://`):** double-click the file.

- **Expect:** Webbase loads immediately — sidebar nav, screen
  content, branding palette. No unlock screen.
- **Network:** only the optional /version freshness check (which
  fails open — page works if Webspinner is unreachable).
- **IndexedDB:** Chrome and Safari allow it on `file://` from
  user-opened files in most configurations. If your data doesn't
  persist after closing the tab, it's the browser, not the file.

**3c-ii. Hosted:** drag the file onto netlify.app's deploy area, or
`python3 -m http.server` in the Downloads folder and open
`http://localhost:8000/<file>`. Same UI, plus IndexedDB always
persists on a real origin.

**3c-iii. Air-gapped:** disconnect from the network entirely, then
double-click the file. The Webbase still runs — only the freshness-
check ping fails silently. This is the load-bearing demonstration of
patron sovereignty.

### 3d. Freshness check

1. Open a downloaded standalone in any browser.
2. Wait ~3 seconds after the page loads.
3. **Verify:** if a newer version was published since the file was
   downloaded, a bottom-anchored banner appears: "A newer version
   (v<N+1>) is published. You're running v<N>. [Download new
   version] [Keep this one]".
4. If no newer version exists, banner is silent — no UI flicker.

### 3e. Verify download was logged

1. Re-open the Your Work modal in try.
2. **Verify:** the row's tally has incremented:
   `v1 · Nd ago · 1 use` or similar. The Account status band's
   download total bumps by 1.

## 8a. Renderer extraction — now complete (closes #58)

The downloaded `.html` is now **truly** self-contained — no CDN
references, no SvelteKit chunks needed. The renderer is compiled
into a 53 KB IIFE bundled alongside the bundle JSON and the CSS,
totaling ~67 KB per file (21 KB gzipped). Verified end-to-end in
headless Chromium opening from `file://`: zero network requests
beyond the optional freshness ping, zero JS errors, full UI renders.

How it works:

- `loom/src/lib/runtime/WebbaseRuntime.svelte` is the renderer
  component. Same file serves `/run/<code>` (hosted) and the
  standalone download.
- `vite.standalone.config.ts` builds it as an IIFE library.
- `deploy-loom` runs `pnpm build:standalone` after `pnpm build`.
- `/app/<code>/standalone` reads the built artifacts, inlines bundle
  JSON + CSS + JS into one HTML, streams it as an attachment.

What this unlocks (and you should mentally model as already true):

- Air-gapped use cases: clinics, classroom kits, field crews.
- Email-the-file federation between patrons. Recipient runs the
  file without ever signing in to Webspinner.
- Bundles outlive the Foundation. If Webspinner shuts down, every
  downloaded file keeps working.
